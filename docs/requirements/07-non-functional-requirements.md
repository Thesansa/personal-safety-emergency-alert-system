# Non-Functional Requirements

## Usability
- The system shall provide a simple and intuitive user interface suitable for use during high-stress situations.

## Performance
- The system shall send emergency notifications within an acceptable response time after SOS activation.

## Security
- The system shall protect user data using authentication and authorization mechanisms.
- The system shall ensure secure storage of location and alert data.

## Reliability
- The system shall maintain consistent alert delivery even under poor network conditions where possible.

## Availability
- The system shall be available for use at all times except during scheduled maintenance.
