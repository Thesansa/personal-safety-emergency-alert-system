# Functional Requirements
-The system shall allow users to register and manage personal accounts. 

-The system shall allow users to configure trusted contacts.  

-The system shall provide a quick SOS activation mechanism for emergency situations.  

-The system shall capture and transmit the userâ€™s real-time location during an emergency.  

-The system shall notify trusted contacts when an SOS alert is triggered.  

-The system shall initiate a countdown timer after an SOS alert is activated.  

-The system shall automatically escalate the alert if the user does not respond within the defined time period.  

-The system shall allow users to cancel an active SOS alert.  

-The system shall record and store emergency alert events securely.  

-The system shall allow trusted contacts to acknowledge received alerts.

# Non-Functional Requirements
## Usability
The system shall provide a simple and intuitive user interface suitable for use during high-stress situations.  

## Performance
The system shall send emergency notifications within an acceptable response time after SOS activation.  

## Security
The system shall protect user data using authentication and authorization mechanisms.  
The system shall ensure secure storage of location and alert data.  

## Reliability
The system shall maintain consistent alert delivery even under poor network conditions where possible.  

## Availability
The system shall be available for use at all times except during scheduled maintenance.
